TECHNICAL SPECIFICATIONS

Data Mining, Model Architecture, and Evaluation Mechanics | Current Patch: 26.14

1. Data Mining Pipeline

Match data is mined using a custom Breadth-First Search (BFS) crawler connected to Riot Games' official API via riotwatcher. The crawler traverses match histories across Platinum+ Ranked Solo Queue games to build a dataset of over 70,000+ completed match drafts (Patch 26.14).

Champions are vectorized into position-aware role tokens (Top, Jungle, Mid, Bot, Support) for both Blue and Red teams. Because champions are mapped dynamically by name strings rather than hardcoded static stats, retraining updates win rates and counter relationships automatically per patch cycle.

2. Deep Learning Architecture & Division of Labor

The prediction engine uses a multi-stage hybrid design where each architectural component targets a specific aspect of League drafting:

  • WIDE Google Wide Path (Base Win Rate Memorization): A linear model path that directly tracks raw, role-specific base champion win rates across ranked solo queue.
  • GCN Graph Convolutional Layer (Relational Topology Prior): Computes pairwise synergy and counter-pick adjacency matrices across all 170+ champions. Through graph message passing, it injects topological priors to stabilize predictions for low-sample-size and niche champions.
  • ATTENTION Multi-Head Self-Attention (5v5 Composition Interplay): A Transformer attention block that processes full 10-champion sequences to compute complex 5v5 composition interplay, damage distribution, and team-wide synergy.

3. Snake-Draft Masked Partial State Training

To support accurate win probability predictions during active champion select (from 1 pick up to 10 picks), training data is augmented across all official Ranked Solo Queue draft stages:

Draft Order: B1 → R1, R2 → B2, B3 → R3, R4 → B4, B5 → R5

Unpicked champion slots are masked using key-padding masks and masked mean-pooling, allowing the neural network to calculate clean, non-distorted expected win rates at any turn of champion select.

4. Training & Optimization

The network is optimized using Binary Cross Entropy Loss with the Adam optimizer. Regularization is enforced via L2 weight decay ($\alpha$) and Early Stopping on validation loss to prevent overfitting.

⚠ Off-meta picks (e.g. Ashe Jungle) are underrepresented in training data so you may see non-sensical predictions in such cases. Predictions become progressively more accurate as more champions are locked in. Not affiliated with or endorsed by Riot Games.
